  • JavaScript is a "dynamic-typed" variable.
    • Use "var" to declare a variable rather than specifying the type at declaration.
    • You can change the type of a variable at any point.
    • a = 1; b = "1";
      a = = b > true
      (JS will try to "normalize" the types)
      a = = = b > false
      (JS will not normalize the types)
    • Types available:
      • 'string' || "'string'" || '"string"'
      • [array] (items can be different types)
        • c = ['a', 2, 2.5, [1,2,3]]
      • bool
      • null
      • undefined
        • Common error to get. (forgot to define a variable)
      • object
        • object in C# is the class that all other objects derive from (hidden base class)
        • object in JS is a type with Key/Value pairs as properties. Properties can be any type - even other obj.
          • objName = { Name: 'Kristin', Role: 'Student', City: 'Greenville' }
          • objName.Name returns "Kristin"
          • Add a property at any time with objName.NewProp = 'MyProp'
  • Semicolon is not a statement terminator but a statement separator. Not required if statements are on separate lines but BP is to use it anyways.
